# Backend Refactoring To-Dos

## Multi-label Classification Conversion (COMPLETED)
The backend now successfully trains and serves a multi-label classification model.

> *Note: `/api/analyze` has been updated to use the fine-tuned DistilBERT model instead of the keyword fallback. The model was trained on ~950 synthetic rows (mixed strict and relaxed trigger-word caps generated by local Llama 3.1 8B) and achieves a 41.5% Macro F1.*

**Completed Steps:**
- [x] **Dataset Generation**: Updated scripts and generated multi-label rows using a robust sanity-checked pipeline.
- [x] **Fine-tuning Notebook / Script**: Updated to use `problem_type="multi_label_classification"` and `BCEWithLogitsLoss`. Implemented pos_weight balancing and per-class threshold sweeping.
- [x] **Evaluation Metrics**: Updated `eval_utils.py` and evaluation scripts to compute multi-label F1 (micro/macro/per-label) correctly.

## Known Limitations for Future Retraining
- **Under-trained "No Distortion" Class**: The model has a known bias toward over-flagging distortions on neutral text, traceable to the under-representation of the negative class during synthetic data generation. 
  - *Concrete Example*: The strictly neutral sentence `"Today was an okay day, nothing special happened."` incorrectly triggers `Magnification and Catastrophizing (51.6%)`. This is a textbook false positive that must be added as a core benchmark for any future dataset expansion.
